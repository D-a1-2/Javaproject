## 组合 vs 继承

本次实验选择继承，因为 Circle、Rectangle、Triangle 
与 Shape 之间存在明确的 is-a 关系：圆形是一种图形，
矩形是一种图形，三角形是一种图形，符合继承的使用条件。

如果两个类之间只是"代码相似"而没有真正的父子关系，
应该使用组合而非继承。典型反例是 Java 早期的 
Stack extends Vector —— 栈并不是向量，
只是恰好可以借助向量实现，这种情况用组合更合适。

判断原则：能通过 is-a 检验的用继承，否则用组合。
## 实验反思

本次实验最大的收获是理解了抽象类的作用——
通过强制子类实现 getArea()，保证了代码结构的统一性，
不会出现某个图形忘记写面积计算的情况。

对多态的理解也更深了一些：ShapeUtil.printArea() 
只写了一个方法，却能处理三种不同图形，
以后新增图形类只需继承 Shape，工具类完全不用改动。

不足之处是对 abstract 关键字的用法还不够熟练，
实验过程中依赖 AI 辅助较多，后续需要加强独立编写的能力。
